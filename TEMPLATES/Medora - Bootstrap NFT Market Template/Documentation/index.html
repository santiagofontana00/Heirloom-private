Intro

Introduction

Medora is a NFT ( non-fungible token ) Digital Asset Marketplace Html Template built on Bootstrap5, HTML5, CSS3, JavaScript. medora is built on the platform of binance smart chain, focusing on NFT, and ease to Create, Buy, Sell, Auction and use NFTs. This is a nice and awesome template that can be used to build any virtual items,crypto Art markets, digital assets biding, digital marketplace, for crypto collectibles and non-fungible tokens.

The template is built with latest Bootstrap stable version 5.0.2, following the new web design/development trends, to attract your target users and encourage them to subscribe into your newsletter.

Getting started

Getting started

This template uses Bootstrap 5 CSS framework, a powerful tool to help you building great websites in a short time.

Template's directory structure

  • HTML files - HTML files are located in the root folder. These are index.html, index-pr.html, etc.
  • /css - CSS files of main template stylesheets and a custom stylesheet for your changes.
  • /pug - pug.js files of main template markup for more customization & easier modification.
  • /img - Images for demo purposes. Mostly replaced by placeholder images in the download archive.
  • /js - Javascript files
  • /scss - SCSS source files for easier customization
  • /vendor - Contains all 3rd party plugins used for this template
  • credits.txt - Text file with all attributions of using plugins & assets.

stylesheets

CSS Files

Theme main stylesheet

The main CSS file is style.defaul.css with its minified version style.default.min.css and the source map version style.default.min.css.map .These files includes the Bootstrap 5 source styling besides our custom style, so, ideally it's all in one stylesheet.

Custom CSS code

I suggest placing your changes and CSS declarations in custom.css file. This way, you will keep your code separated and you can update the original template stylesheet if an updated is released.

SCSS

SCSS Files

What is SASS and SCSS

This template contains its SCSS source codes too. These can be very handy when you would want to create your own custom colour variant, create a new button style and similar situations.

If you are completely new to SASS, check out their official website and download a compiler for your OS. SASS uses a bit different syntax than CSS but you should not have a problem working with it if you understand CSS.

File structure

he easies way to change theme colours, fonts and any CSS rules in general is by editing the SCSS files.

In the SCSS directory, there is a main file style.default.scss . In these files, we just define which of the predefined colours will be used and the rest is done by core.scss file. It takes control of all the necessary module includes. The modules (blog, pages, etc.) are located in the modules subdirectory.

scss
_core.scss (core file handling all the includes)
style.default.scss (main file)

-- <theme> (includes all partials included in the main stylesheets - style.default.scss, etc.)

---- _3rd-party.scss - styling of 3rd party plugins used in the template
---- _general.scss - generic styling of the template -buttons, header, footer, sections..
---- _mixins-custom.scss - my custom mixins
---- _utils - custom bootstrapious utils
---- _preloader - styling of template preloader
---- _theme - including all partials
---- _variables.scss - file based on the Bootstrap 5 variables file

-- <user> (includes all partials prepared for you to apply their own styles)

---- _user-variables -  the place for your variable overrides
---- _user - this is the place for your components / CSS

-- <bootstrap> (includes Bootstrap 5 sources files)

JavaScript

JavaScript Files

The javascript files includes are:

  • /countdown.js - Lightweight vanilla js plugin for setting up auctions' counters
  • /particles.json - Configuration of Particles.js plugin for desktop viewport
  • /particles.mb.json - Configuration of Particles.js plugin for mobile viewport
  • /theme.js - Theme main js file that holds all configurations of all used scripts.
jQuery dependency?

This template doesn't require jQuery as a dependency for javascript script, all code is written in plain javascript. For example, this a snippet for removing preloader after page loading

theme.js
/* =====================================================
        REMOVE PRELOADER AFTER DOM CONTENT LOADED
===================================================== */
document.body.style.overflow = "auto";
document.body.style.overflowX = "hidden";
const preloaderInner = document.querySelector(".preloader-inner");

if (preloaderInner) {
    preloaderInner.style.opacity = 0;
    setTimeout(function () {
        preloaderInner.parentElement.style.display = "none";
    }, 500);
}
